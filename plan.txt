listener.py
import speech_recognition as sr

recognizer = sr.Recognizer()

def listen():
    with sr.Microphone() as source:
        print("🎧 Listening...")
        recognizer.adjust_for_ambient_noise(source)
        audio = recognizer.listen(source)

    try:
        text = recognizer.recognize_google(audio)
        print("🗣 You said:", text)
        return text.lower()
    except sr.UnknownValueError:
        return ""
    except sr.RequestError:
        return ""

import wikipedia

def search_wikipedia(query):
    try:
        query = query.replace("who is", "").replace("what is", "")
        result = wikipedia.summary(query, sentences=2)
        return result
    except:
        return "Sorry, I couldn't find that."

🏁 7️⃣ Main Runner (Wake-Word + Loop)
main.py
from listener import listen
from speaker import speak
from intents import handle_intent
from config import WAKE_WORD

def run_alexa():
    speak("Alexa online. Say Alexa to wake me up.")

    while True:
        text = listen()

        if WAKE_WORD in text:
            speak("Yes, how can I help you?")
            command = listen()

            if "exit" in command or "stop" in command:
                speak("Goodbye!")
                break

            response = handle_intent(command)
            speak(response)

if __name__ == "__main__":
    run_alexa()

📘 PROJECT SYNOPSIS
Title of the Project

Design and Development of an Intelligent Voice Assistant (Alexa Clone) Using Python

Abstract

With the rapid advancement of Artificial Intelligence and Natural Language Processing, voice-controlled virtual assistants have become an integral part of modern human–computer interaction. This project focuses on the design and development of an intelligent voice assistant inspired by Amazon Alexa, capable of understanding human voice commands and responding intelligently through speech.

The system integrates speech recognition, text-to-speech synthesis, intent classification, and modular skill execution to provide real-time assistance. It supports wake-word detection, conversational interaction, web-based knowledge retrieval, system information access, and extensible skill-based functionalities. The proposed solution is lightweight, cost-effective, and runs efficiently on consumer-grade hardware without the need for specialized GPUs, making it suitable for educational and real-world applications.

Problem Statement

Traditional interaction with computers relies heavily on keyboards and touch-based inputs, which may not be efficient or accessible in all scenarios. There is a growing need for natural, hands-free, and voice-driven interfaces that can assist users in retrieving information, performing system operations, and interacting with digital services seamlessly.

Commercial voice assistants like Alexa and Google Assistant are proprietary, costly, and cloud-dependent, limiting customization and learning opportunities for students. Therefore, there is a need to develop an open, customizable, and locally runnable voice assistant that demonstrates the core working principles of modern AI-based virtual assistants.

Objectives of the Project

The primary objectives of this project are:

To design a voice-controlled assistant capable of understanding and processing human speech.
To implement wake-word detection for activating the assistant.
To convert speech input into text using speech recognition techniques.
To generate natural voice responses using text-to-speech synthesis.
To identify user intent and route commands to appropriate services.
To develop a modular skill-based architecture similar to commercial voice assistants.
To ensure extensibility for adding future AI-powered features.
To build a fully functional system that runs on local machines without heavy hardware requirements.
Scope of the Project

The scope of the project includes:

Voice-based command execution
Conversational interaction with users
Information retrieval using web APIs
System-level queries such as time, date, and device information
AI-ready architecture for chatbot integration
Skill-based modular expansion

The project does not aim to replicate all proprietary features of commercial assistants but focuses on implementing the core intelligence and architectural concepts behind them.

System Architecture

The system follows a layered and modular architecture:

Input Layer – Captures voice input via a microphone.
Speech Recognition Layer – Converts speech into text.
Wake-Word Detection Module – Activates the assistant upon detecting a predefined keyword.
Intent Processing Layer – Analyzes user commands and determines intent.
Skill Execution Layer – Executes corresponding skills such as time query, system info, or web search.
Response Generation Layer – Produces a textual response.
Text-to-Speech Layer – Converts text responses into audible speech output.

This architecture ensures scalability, maintainability, and ease of feature expansion.
